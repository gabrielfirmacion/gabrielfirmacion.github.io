Ruby Classes VS JavaScript Objects

What is the difference?

Ruby hashes and JavaScripts Objects have the same concept and they look very similar as seen below.

JavaScript Object
var car = {
___type:"Fiat",
___model:500,
___color:"white"
};

Ruby Hash
car = {
___“type” => “Fiat”,
___“model” => 500,
___“color” => “white”,
}

They both have name-value pairs. In Ruby, the pairs are called “key-value” pairs while in JavaScript these are called “properties.” If you want the value of “Fiat” to appear, in Ruby, you will type car[“type”] #”Fiat”. While in JavaScript you can call it the same in Ruby or by typing car.type; // “Fiat”.

In Javascript you can add properties by just doing the following: car.size = “small”; In Ruby you can add key-value pairs by typing car[“size”] = “small”

As you can see, they are very similar in terms of they look and their use. However, syntax is still different. If you mix them up accidentally they will not work.
